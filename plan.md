# Sanskrit Knowledge Ecosystem Website Plan

## Goal

Build the website as a structured digital portal for the **Sanskrit Knowledge Ecosystem** project, covering all 15 required topics in a way that is:

- easy to navigate
- visually organized
- presentation-friendly for PPT and report use
- credible through citations and institute links

The website should not feel like one long assignment dump. It should feel like an academic knowledge portal with clear sections, repeatable page patterns, and strong source visibility.

---

## Recommended Site Structure

Use **6 main content pages + home page** instead of creating 15 separate pages.

### Main Pages

1. `index.html`
   Project homepage and ecosystem overview

2. `pages/indian-institutions.html`

3. `pages/global-programs.html`

### Optional Supporting Pages

  master bibliography / all references in one place
- `pages/report.html`
  downloadable report, PPT, and appendix links

## 6. Methodology Page

Topic covered:

- Topic 15: Website development and research process

This page is essential because the assignment explicitly asks for the process used to find the data.

### Sections

#### 1. Research Objective

Explain:

- what the project set out to document
- what categories were included

#### 2. Research Workflow

Use a step-by-step format:

1. Read the assignment and identify all 15 topics
2. Group related topics into website sections
3. Identify trusted sources
4. Visit institute and organization websites
5. Extract program, faculty, archive, journal, award, and scholarship data
6. Cross-check information across official pages
7. Save institute links and citations
8. Organize the material into page-wise sections
9. Design and build the website structure
10. Prepare report and PPT based on website findings

#### 3. Source Types Used

Examples:

- official university websites
- department pages
- organization websites
- archive catalogues
- digital library portals
- journal websites
- government / cultural ministry portals
- scholarship and fellowship announcements
- official YouTube channels or structured educational channels

#### 4. Data Verification Method

Explain:

- preference for official sources
- use of multiple sources for confirmation
- checking program availability and dates
- recording direct institutional links

#### 5. Content Organization Method

Explain:

- why the website was divided into major pages
- how repeated templates were used
- how tables/cards improve comparison

#### 6. Output Deliverables

Mention:

## Source and Citation Rules

These are important for credibility and for the assignment requirement.

- Every institution mentioned should include its official link.
- Every page should end with a `Sources` section.
- Use official websites first whenever possible.
- If secondary sources are used, keep them limited and clearly supportive.
- Record the access date for online references.

## Final Recommendation

The best structure for this project is:

- one homepage for overview
- one page per major category
- one methodology page for the research process

This is the most balanced option because it is:

- easier to build than 15 separate pages
- easier to navigate than one giant page
- better for presentation and grading
- easier to expand later with more institute data
